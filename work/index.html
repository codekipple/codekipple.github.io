Carl Hughes

Front end developer based in Bristol. I work with teams and companies to create performance driven accessible sites.

Core Skills

CSS architecture

I am extremely interested in CSS architecture. It's surprising just how difficult CSS architecture can be, particularly on larger sites/apps which have multiple developers adding and editing CSS.

I am constantly working towards improving my CSS architecture. I currently use a lot of ideas i have gleaned from Nicole Sullivan, Nicolas Gallagher and Harry Roberts.

Front end JavaScript

I like JavaScript that does one thing or group of things well, preferably packaged using the commonJS module pattern. I am comfortable using both commonJS and AMD JavaScript modules.

Web markup

I care a lot about markup. I am constantly reviewing the mark up I use and why.

The HTML5 Doctor site is a brilliant resource which I consult often as parts of the HTML specification changes over time.

I care about accessibility and where appropriate I will use ARIA to make your content available to more people.

I regularly use both RDFa and Microdata to provide structured data for services like search engines to make use of.

Front end ops

I am happy JavaScript based Task managers to accomplish front end ops.

Front end ops can help you minify and optimise assets and automate tasks which otherwise would have been done by a human.

More specifically i have used both Grunt and Gulp but I am more than happy to use whatever tool your team uses.

If you team is yet to embrace these types of tools but would like to I can provide help, advice and training to get these tools working for you.

Site/app deployment

A smooth and easy deployment method is needed to encourage rapid development of your site.

I have experience of using programs such as Capistrano to make deployment as simple as a one liner in the console.

If your team are still deploying manually I can help setup better work flows so your team can concentrate on building rather than deploying.
